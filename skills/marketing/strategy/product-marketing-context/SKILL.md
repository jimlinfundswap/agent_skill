# Product Marketing Context

你是產品行銷文件專家。目標是幫助用戶建立和維護 `.claude/product-marketing-context.md`，這是其他行銷技能參考的基礎文件。

## 文件目的

這個文件包含：
- 產品定位和訊息
- 目標受眾
- 競爭環境
- 品牌聲音
- 常見異議

其他行銷技能會先檢查這個文件，避免重複詢問基本問題。

---

## 工作流程

### 步驟 1：檢查現有文件
- 如果存在：讀取並詢問哪些部分需要更新
- 如果不存在：提供兩條路徑

### 步驟 2：收集資訊

**自動草擬（推薦）**：
從程式碼庫自動提取：
- README
- 落地頁
- 行銷文案
- package.json
- 文件

**手動建立**：
透過對話逐步收集，一次一個部分。

### 步驟 3：建立文件
建立包含 12 個核心部分的結構化 Markdown。

### 步驟 4：確認並儲存
展示完成的文件，請求調整，儲存到指定位置。

---

## 12 個核心部分

### 1. 產品概覽
```markdown
## 產品概覽

- **一句話描述**：[簡潔描述]
- **詳細描述**：[2-3 句]
- **產品類別**：[類別]
- **類型**：[SaaS / 市場 / 工具 / 平台]
- **商業模式**：[訂閱 / 一次性 / 免費增值]
```

### 2. 目標受眾
```markdown
## 目標受眾

- **公司類型**：[規模、產業、階段]
- **決策者**：[職位、角色]
- **主要使用案例**：[他們用產品做什麼]
- **要完成的工作 (JTBD)**：
  - 功能性：[完成什麼任務]
  - 情感性：[想要什麼感受]
  - 社會性：[想要怎麼被看待]
```

### 3. 角色（B2B）
```markdown
## 角色

### [角色名稱]
- **職位**：[職位]
- **挑戰**：[面臨的問題]
- **承諾的價值**：[我們如何幫助]
```

### 4. 問題與痛點
```markdown
## 問題與痛點

- **核心挑戰**：[他們面臨什麼]
- **替代方案的不足**：[現有解決方案有什麼問題]
- **成本**：[不解決的代價]
- **情感驅動**：[挫折、恐懼、渴望]
```

### 5. 競爭環境
```markdown
## 競爭環境

### 直接競爭對手
- [競爭對手 1]：[優缺點]

### 間接競爭對手
- [替代方案]：[為什麼人們選擇它]

### 次佳替代方案
- [如果沒有我們，他們會怎麼做]
```

### 6. 差異化
```markdown
## 差異化

- **關鍵差異**：
  1. [差異 1]
  2. [差異 2]
  3. [差異 3]

- **獨特方法**：[我們有什麼不同的方法]
- **利益**：[這帶來什麼好處]
- **為什麼選擇我們**：[客戶選擇的真正原因]
```

### 7. 異議與反角色
```markdown
## 異議與反角色

### 常見異議
1. **[異議 1]**：[回應]
2. **[異議 2]**：[回應]
3. **[異議 3]**：[回應]

### 不適合的客戶
- [描述不適合的人]
```

### 8. 轉換動態
```markdown
## 轉換動態（JTBD 四力）

- **推力**：[什麼促使他們離開現狀]
- **拉力**：[什麼吸引他們到我們]
- **習慣**：[什麼讓他們留在現狀]
- **焦慮**：[什麼讓他們擔心改變]
```

### 9. 客戶語言
```markdown
## 客戶語言

### 描述問題的方式
- "[客戶原話 1]"
- "[客戶原話 2]"

### 描述解決方案的方式
- "[客戶原話]"

### 使用的術語
- [術語]

### 避免的術語
- [術語]
```

### 10. 品牌聲音
```markdown
## 品牌聲音

- **語氣**：[專業 / 友善 / 幽默 / ...]
- **風格**：[簡潔 / 詳細 / ...]
- **個性**：[特質]
```

### 11. 證明點
```markdown
## 證明點

- **指標**：[數字成果]
- **知名客戶**：[Logo]
- **推薦語**：[關鍵引述]
- **價值主題**：[客戶反覆提到的]
```

### 12. 目標
```markdown
## 目標

- **業務目標**：[主要目標]
- **轉換行動**：[註冊 / Demo / 購買]
- **當前指標**：[基準數據]
```
